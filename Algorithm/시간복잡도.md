# 시간복잡도



keyword : Algorithm,  시간복잡도



## 시간복잡도란?

입력값과 문제를 해결하는데 걸리는 시간과의 상관 관계를 말한다.

입력값이 2배로 늘어났을 때 문제를 해결하는 데 걸리는 시간은 몇 배로 늘어나는지 보는 것이다.



## 시간복잡도 예시 - 최댓값 찾기



만약 다음과 같은 6개의 숫자가 있고 이 중 최댓값을 찾아야 한다면 방법에 따라 시간복잡도가 어떻게되는지 보자. 각 코드의 라인이 실행될 때마다 1번의 연산이 된다고 생각하고 계산을 해보자.

```python
input = [3, 5, 6, 1, 2, 4]
```



1. 첫번째 해결 방법

   - 각 숫자마다 모든 다른 숫자와 비교해서 최대값인지 확인하기

     ```python
         for num in array:              # array 의 길이만큼 아래 연산이 실행
             for compare_num in array:  # array 의 길이만큼 아래 연산이 실행
                 if num < compare_num:  # 비교 연산 1번 실행
                     break
             else:
                 return num
     ```

   - for 문을 2번 돌려 모든 숫자끼리 비교를하니 `N x N` 만큼의 시간이 걸렸다고 볼 수 있다.

     

2.  첫번째 해결 방법

   - 리스트를 하나씩 돌면서 최댓값 변수에 저장된 값과 비교하는 방법

     ```python
     		for num in array:      # array 의 길이만큼 아래 연산이 실행
     		    if num > max_num:  # 비교 연산 1번 실행
     		        max_num = num  # 대입 연산 1번 실행
     ```

   - for문을 한번만 돌리고 비교연산과 대입연산을 1번씩 하므로 수식으로 표현하게 되면 `2N+1`로 표현할 수 있을 것이다.



위의 두 방법을 비교해보면 N의 값이 커짐에 따라 연산이 기하급수적으로 늘어나는 것을 확인할 수 있다.



| N의 길이 | N^2       | 2N+1  |
| -------- | --------- | ----- |
| 1        | 1         | 3     |
| 10       | 100       | 21    |
| 100      | 10000     | 201   |
| 1000     | 1000000   | 2001  |
| 10000    | 100000000 | 20001 |



따라서 이를 참고하여 코드를 짤 때 더 연산이 적고 효율이 좋은 즉, 시간 복잡도가 낮은 알고리즘을 선택하여 코드를 짜도록하자.
